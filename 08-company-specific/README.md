# Company-Specific Strategy

## Purpose
Translate the general six-month plan into company-focused final preparation for Google, Microsoft, Amazon, Uber, Flipkart, Atlassian, and LinkedIn.

## Study Steps
- Read interview-patterns.md first.
- Pick two target companies by month 5.
- Run company-specific mocks in the final 2 weeks before each loop.
- Verify current process with recruiter because round mixes change.

## How To Use
- Month 5: choose target companies and start focused mocks.
- Month 6: use final 2-week plans before each actual loop.
- Always combine company docs with `06-interview-tracker/mock-feedback.md`.

## Strategy Rule
Do not change your whole preparation for one company. Adjust emphasis: Google coding clarity, Amazon LPs, Flipkart LLD, Atlassian code design, Microsoft collaboration/customer focus, Uber scale and execution, LinkedIn graph/feed/search systems.

## Interview Questions
- Which target company changes your mock mix the most?
- Which behavioral stories map to that company?
- What should you verify with the recruiter?

## Common Mistakes
- Using outdated crowd-sourced loop structures as guaranteed truth.
- Company cramming before core readiness.
- Ignoring behavioral calibration until the final week.

## Self-Check
- [ ] I have two target companies selected.
- [ ] I can name the highest-weight round for each.
- [ ] I have completed at least two company-style mocks.

## Practical Example
Example: For Flipkart, shift one weekly design block to LLD/machine coding from month 5; for Amazon, turn every project and incident story into STAR with Leadership Principle mapping.
