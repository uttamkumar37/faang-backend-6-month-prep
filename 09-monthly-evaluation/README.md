# Monthly Evaluation Gates

## Purpose
Define hard monthly readiness gates so the six-month plan does not become passive content consumption.

## Study Steps
- At the end of each month, open the matching gate.
- Copy scores from the tracker scorecards.
- Failing a gate triggers a recovery week.
- Month 6 uses final readiness, not just completion.

## Gate Files
| Month | File | Theme |
|---|---|---|
| 1 | month-1-gate.md | Fundamentals reset |
| 2 | month-2-gate.md | Java, concurrency, LLD start |
| 3 | month-3-gate.md | Distributed systems and production debugging |
| 4 | month-4-gate.md | Advanced design, DP, AI/RAG |
| 5 | month-5-gate.md | Interview simulation and company strategy |
| 6 | month-6-final-readiness.md | Final SDE-II readiness |

## Scoring Rule
Score each gate out of 100. Passing means 75+ overall and no critical area below the stated minimum.

## Interview Questions
- What evidence proves you can move to the next month?
- Which low score needs a recovery week?
- What category is most likely to cause downlevel?

## Common Mistakes
- Advancing because the calendar advanced.
- No mock interview evidence.
- Counting project work without demo or design output.

## Self-Check
- [ ] I can produce score evidence for every category.
- [ ] I know pass/fail criteria for this month.
- [ ] Recovery work is scheduled if needed.

## Practical Example
Example: If month 2 LLD is 20/100 because no design was attempted, do not postpone LLD to the final month; run two machine-coding drills before month 3.
