# Company Interview Patterns

## Purpose
Summarize cross-company round patterns and how to prepare without chasing rumors.

## Study Steps
- Verify the exact loop with the recruiter for your location, level, and team.
- Map your weekly practice to the round types listed here.
- Run two company-style mocks before applying or before onsite.
- Prepare behavioral stories using that company's evaluation language without forcing buzzwords.

## Common Round Types
| Round | What is tested | Preparation artifact |
|---|---|---|
| Coding | DSA, correctness, communication | DSA scorecard and timed mocks |
| System design | Architecture, scale, trade-offs | Design scorecard and written designs |
| LLD/code design | Object modeling, APIs, extensibility | LLD docs and Java skeletons |
| Backend deep dive | JVM, Spring, DB, Kafka, Redis, incidents | Java backend scorecard and debugging playbooks |
| Behavioral/manager | Ownership, collaboration, judgment | Story bank and STAR drafts |
| Project deep dive | Real ownership and technical depth | Project tracker and resume bullets |

## Preparation Mix By Company Signal
- Coding-heavy: increase timed DSA and communication practice.
- LLD-heavy: implement Java machine-coding skeletons with tests.
- Design-heavy: run 45-minute design mocks with reliability and observability.
- Behavioral-heavy: prepare STAR stories with measurable outcomes and failure reflection.

## Interview Questions
- What signals distinguish SDE-II from SDE-I?
- How do companies combine behavioral evaluation with technical rounds?
- What is the safest way to handle uncertain loop structure?

## Common Mistakes
- Assuming all FAANG interviews are identical.
- Practicing only questions, not communication.
- Not preparing project deep dives as technical evidence.

## Self-Check
- [ ] I can classify every scheduled round into a preparation artifact.
- [ ] I know my weakest company-specific signal.
- [ ] I have a 2-week plan for each target company.

## Practical Example
Example: If your loop has coding, system design, and manager rounds, do not add random new topics; run one coding mock, one design mock, and rehearse three manager stories.
