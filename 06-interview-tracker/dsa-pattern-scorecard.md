# DSA Pattern Scorecard

## Purpose
Score coding readiness by pattern so practice is not biased toward comfortable topics.

## Study Steps
- Use this file during the Sunday review block.
- Fill concrete numbers first: solved count, mocks, defects, shipped project work.
- Mark pass or fail using the criteria in the file, then pick one recovery action.

## Scoring Scale
| Score | Meaning |
|---:|---|
| 0 | Cannot recognize pattern |
| 1 | Recognizes after hint but cannot implement cleanly |
| 2 | Solves easy, struggles on medium variants |
| 3 | Solves medium with occasional bugs in 35-45 minutes |
| 4 | Solves most mediums in 25-30 minutes with tests |
| 5 | Explains variants, proves correctness, handles follow-ups |

## Pass/Fail Criteria
- Pass for SDE-II loop: no core pattern below 4 and DP/graphs/trees at least 3.5.
- Fail: any core pattern below 3, or medium problems regularly exceed 40 minutes.

## Interview Questions
- How would you classify this problem before coding?
- What invariant makes your solution correct?
- What is the brute force and what changes to optimize it?

## Common Mistakes
- Practicing random problems without pattern tags.
- Ignoring time-to-first-correct-code.
- Not re-solving failed mediums without looking.

## Self-Check
- [ ] I can identify the pattern in under 2 minutes for common mediums.
- [ ] I can explain correctness before code.
- [ ] I have at least 3 evidence problems for every score 4 pattern.

## Practical Example
Example: Binary search score 4 requires solving rotated search, first/last position, minimum capacity, and answer-space variants without off-by-one debugging longer than 5 minutes.
