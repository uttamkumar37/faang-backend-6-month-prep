# Mock Interview Feedback Template

## Purpose
Capture feedback from coding, system design, LLD, Java backend, and behavioral mocks in a form that produces concrete repairs.

## Study Steps
- Use this file during the Sunday review block.
- Fill concrete numbers first: solved count, mocks, defects, shipped project work.
- Mark pass or fail using the criteria in the file, then pick one recovery action.

## Interview Questions
- Where did the interviewer need to rescue you?
- Which follow-up exposed shallow depth?
- What would make this a strong hire signal next time?

## Common Mistakes
- Recording only the score and not the evidence.
- Treating communication feedback as personality feedback instead of a skill.
- Not assigning due dates to repairs.

## Self-Check
- [ ] Every mock produces at least one specific drill.
- [ ] I can compare the last three mocks and see improvement.
- [ ] Borderline rounds are repeated within two weeks.

## Practical Example
Example: Design mock feedback: 'Missed idempotency and DLQ in payment flow' becomes a repair drill: redo payment system design with duplicate charge, provider timeout, and reconciliation scenarios.
