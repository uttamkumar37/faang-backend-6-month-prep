# Month 1 Gate

## Purpose
Evaluate month 1 progress against SDE-II backend readiness expectations.

## Study Steps
- Fill the score honestly before starting the next month.
- Use evidence from trackers, not memory.
- If below pass, execute the recovery plan before adding new scope.

## Required Completion
| Category | Requirement |
|---|---|
| DSA count | 35-45 total, focused on arrays, strings, hashing, two pointers, sliding window, binary search |
| System design count | 2 written designs: URL shortener and rate limiter basics |
| Java backend topics | Collections, REST basics, Spring Boot request flow |
| Mock interview count | 2 mocks or timed sets |
| Project progress | Two flagship projects selected; first API skeleton planned |
| Behavioral stories | 2 stories drafted: ownership and ambiguity |
| Extra senior-backend focus | Weekly tracker and mistake log active |

## Pass/Fail Criteria
Pass: 75+ overall, DSA at least 14/20, and mistake log has 10+ entries with repairs. Fail: no mock or no project selection.

## Recovery Plan If Low
Spend 5 days on weak DSA patterns, redo both designs, and draft missing stories before month 2.

## Interview Questions
- What did this month prove in an interview setting?
- Which area has the weakest evidence?
- What repair would most improve next month's score?

## Common Mistakes
- Filling scores without evidence links.
- Treating watched videos as completed skill.
- Ignoring behavioral and project readiness until late.
