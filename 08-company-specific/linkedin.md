# LinkedIn Strategy

## Purpose
Prepare for LinkedIn backend interviews with graph/feed/search scale, Java services, product impact, and collaboration stories.

## Study Steps
- Verify the exact loop with the recruiter for your location, level, and team.
- Map your weekly practice to the round types listed here.
- Run two company-style mocks before applying or before onsite.
- Prepare behavioral stories using that company's evaluation language without forcing buzzwords.

## Official Reference
- https://www.linkedin.com/company/linkedin/life

## Round Structure To Expect
- Recruiter or hiring-manager screen.
- Coding/DSA round or online assessment depending on pipeline.
- System design for SDE-II/Senior backend roles where level requires it.
- LLD, code design, or machine coding in companies/teams that emphasize implementation design.
- Behavioral, leadership, values, or manager discussion.

Exact count and order vary by team, location, and level. Treat recruiter instructions as the source of truth.

## DSA Expectation
- Medium problems should be solved in 25-30 minutes.
- Communicate constraints, brute force, optimized approach, correctness, and tests.
- Prepare arrays, graphs, trees, heaps, intervals, binary search, DP basics, and hash maps.

## System Design Expectation
- Start with requirements and scale.
- Design APIs, data model, services, storage, cache, messaging, reliability, and observability.
- Discuss trade-offs and failure modes.

## Java/Backend Expectation
- Java collections, concurrency, JVM, Spring Boot, SQL/indexes, Kafka, Redis, and production debugging.
- Be ready for practical scenarios: slow API, duplicate events, cache invalidation, DB contention.

## Behavioral Expectation
- Use STAR with metrics.
- Show ownership, ambiguity handling, conflict resolution, mentoring, and production maturity.

## Most Important Topics
Coding, distributed systems, feed/search/recommendation trade-offs, data infrastructure, and cross-functional impact.

## Company-Specific Common Mistakes
- Treating the company doc as a replacement for core readiness.
- Not asking recruiter which rounds are included.
- Preparing stories without measurable impact.

## Final 2-Week Revision Plan
| Day range | Focus |
|---|---|
| Days 1-2 | Coding mocks focused on graphs, heaps, strings, intervals, and clean Java |
| Days 3-5 | LinkedIn-domain designs: feed, profile search, messaging, notifications, connection graph |
| Days 6-7 | Data and ranking backend discussion: fanout, freshness, dedupe, counters, privacy constraints |
| Days 8-9 | Backend depth drills: Kafka streams, caching, search indexing, DB partitioning, observability |
| Days 10-11 | Behavioral stories around product impact, collaboration, and technical leadership |
| Days 12-13 | Full mixed loop and targeted repair from mock feedback |
| Day 14 | Resume/project story polish and recruiter logistics |

## Notes
Use recruiter guidance for exact loop structure; practice LinkedIn-domain systems like feed ranking, profile search, messaging, and notifications.

## Interview Questions
- What round is most likely to reject you for this company?
- Which project story best maps to this company?
- What company-specific system would you design in a mock?

## Common Mistakes
- Over-indexing on rumored questions.
- Skipping recruiter verification.
- Not tuning behavioral examples to company expectations.
